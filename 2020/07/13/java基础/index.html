chapter 11、 异常、断言、日志、调试

由程序错误导致的异常：RuntimeException

程序本身没有问题，但由于向I/O错误这类问题导致的异常属于其他异常。

(未检查异常)派生于RuntimeException的异常包括：需要在程序中进行if判断或提前处理防止用户输入错误：例如在程序中判断是否数组越界，或使用变量是否为空。

  • 类型转换异常
  • 数组访问越界
  • 访问空指针

（已检查异常）不是派生于RuntimeException的异常：(需要在程序中处理或者抛出)

一个方法只需要抛出已检查异常，不需要抛出Error（不可控制）和RuntimeException（避免发生）.

使用异常机制的几个技巧

  1. 异常处理不能代替简单的测试，例如进行出栈操作时查看栈内是否为空用if(!s.empty()) s.pop();而不用异常去处理：

因为通常简单的测试用时会比捕获异常用时少很多。在测试的机器上，调用isEmpty()运行时间为646毫秒，捕获mptyStackException用时21739毫秒。

So, use Exception System when the Exception will occur;

PringStream out;
Stack s;
for (i = 0;i < 100; i++) {
try {
n = s.pop();
} catch (EmptyStackException e) {
//stack was empty
}
try {
out.writeInt(n);
} catch (IOException e){
//problem writing to file
}
}


try {
for (i = 0; i < 100; i++) {
n = s.pop();
out.writeInt(n);
}
} catch (IOException e) {
//problem writing to file
} catch (EmptyStackException e) {
//stack was empty
}
